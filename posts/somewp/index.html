嘿嘿，昨天有几个《简单》的 Misc， 菜菜的我们研究了一整天，今天来写一下 WP。

[WEEK2]ex_word.zip

这是个压缩包，打开，里面是个 Word 文档。

解压出来，用 libreoffice 打开。

嗯… 没什么有价值的信息。

根据我小时候用 winrar 打开过 docx 文件的经验来看， docx 文档本质上也可能是压缩文件。

因此，我们将文件后缀改为zip

$ mv 你抓鲁迅，跟我周树人有什么关系.docx 你抓鲁迅，跟我周树人有什么关系.zip -v

然后进行解压

unzip 你抓鲁迅，跟我周树人有什么关系.zip

得到三个目录，一个文件

打开 word/media 目录，发现两张图片，左边

分别用 binwalk 查看

$ binwalk image1.jpeg 

$ binwalk theme.png

发现 theme.png 有猫腻， binwalk 返回的数据告诉我们它其实是个 zip 压缩文件。

因此，我们试着改它的后缀为 *.zip

$ mv theme.png theme.zip -v

解压这个压缩包

unzip unzip theme.zip

然后得到 flag.xml

用 cat 查看它的内容得到 flag

cat flag.xml

得到一串 emoji 表情:

👥👪👪👚👫👝👲👫🐧🐨👪👖🐨👪👖👫👟🐧👖👫👩👬🐪👖👝🐨👘👞👴

用 emoji 在线解密工具解密获得 flag

解密得到: nssctf{t01s_1s_th0_tru3_f1ag}

至此，此题已解。

二维码里只有黑和白.pdf

一个 pdf 文件，用浏览器打开是空白，表面上并不会得到任何结果。

其实 pdf 本质上也是 zip 压缩

不然可以用 binwalk 看一下，

$ binwalk 二维码里只有黑和白.pdf

我们试试直接把它格式改为 zip

$ mv 二维码里只有黑和白.pdf 二维码里只有黑和白.zip -v

解压，得到一张图片

也许我们会被误导，因为最开始的文件提到了 “二维码”，但是我们用手机却无法扫出来。

后来我去用谷歌识图搜索了一下，发现它确实是一种二维码，叫 MaxiCode.

MaxiCode是由United Parcel Service最初创建和使用的公共域，机器可读的符号系统。它类似于Aztec码或QR码，适用于跟踪和管理包裹的运输，但使用的点排列成六角形网格而不是方形网格。MaxiCode已根据ISO / IEC 16023进行了标准化。

这种码一般是无法用手机扫出来的（这是半小时举着手机得出来的结论）

因此，我们需要去寻找 MaxiCode 的在线解码工具。

用在线工具得出一串 base64 码: bnNzY3Rme0lfaDB0ZV90aGVfUVJjMGRlISEhIX0=

解开这个 Base64 就可以得到答案了。

这里我们用 Python, 这个脚本已经在我们的库里了哦～

# /usr/bin/python
# -*- coding: utf-8 -*-

import argparse
import base64

parser = argparse.ArgumentParser(description="A base64 decoder script made by WeepingDogel for CTF.")
parser.add_argument('-d', '--decode', help="decode some strings.")
args = parser.parse_args()
r = base64.b64decode(args.decode)
print(r)

$ python SimpleTools/base64decode.py -d 'bnNzY3Rme0lfaDB0ZV90aGVfUVJjMGRlISEhIX0='

得到flag: nssctf{I_h0te_the_QRc0de!!!!}'

p02.exe

嘿嘿，表面上看这是个 exe 文件，但想想，咱现在在 kali 上， 是 Linux， 那 Linux 怎么能跑 exe 文件呢？是吧

直接拿 hexdump 来看

$ hexdump -C p02.exe

看到了一句话： This program cannot be run in DOS mode.

意思是，在 dos 模式这个程序跑不了。

我们再往下找找有没有什么信息。

找到了

flag 在这

复制粘贴出来 NSSCTF{oh_you_find_it}

源码.zip

这个最难

首先我们解压压缩包

得到一张图片和一个压缩包

压缩包有加密，因此我们得从这个图片中获得密码。

先 hexdump 吧

$ hexdump key.png -C

找到线索

{GMZTGYJQGM3TAYTDMQYGGMBUGRQWENTYGFRWMMZRMI2DINZYMUYDM===}..{broken password}

broken password 翻译过来是损坏的密码。

我试过用 base64 去解这一串，但是解不出来。

因此，我蒙它是 base32

echo 'GMZTGYJQGM3TAYTDMQYGGMBUGRQWENTYGFRWMMZRMI2DINZYMUYDM===' > flag.txt

base32 -d flag.txt

得到一串 333a0370bcd0c044ab6x1cf31b4478e06

不知道它是什么加密，是 md5 吗？看起来像。

无法解密。

查了一下，md5只有32位字符。

而这一串有33位。

我们堵它是md5,但是多了一位，不知道多出来的是哪一位。

得到的又是图片

用 binwalk 查看

$ binwalk flag1.png

嗯… 到最后还是有个图片隐写。

我们用 dd 命令分离， skip=要分离出的那一项的长度， 我们要分离中间的出来

$ dd if=flag1.png of=flag.png skip=74016 bs=1

又得到一张图片，打开图片显示的就是我们的 flag 啦～！
